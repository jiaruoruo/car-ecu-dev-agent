# 设计模式：MISRA 友好的有限状态机（长期记忆 · 详设/编码召回）

车身域控制（车窗/天窗/尾门/雨刮）普遍采用周期任务驱动的有限状态机：

- 状态用 `enum`，显式赋初值（`= 0u`），避免隐式整型。
- 主调度用 `switch(state)`，**必须含 `default` 分支**（MISRA Rule 16.4），
  default 作为防御式恢复（回到安全态 IDLE）。
- 每个周期先做入参防御（`NULL_PTR` 检查），再做安全相关判定（如防夹）。
- 安全反转/堵转保护等“安全机制”应优先于常规运动判定。
- 标定量（电流阈值、去抖周期、反转行程）用 `#define` 或标定接口，便于产线标定。

防夹典型时序：检测（电流阈值 + 去抖 N 周期）→ 反转固定行程 → 闭锁（BLOCKED）。
相关编码规则见 [[misra_rules]]。
