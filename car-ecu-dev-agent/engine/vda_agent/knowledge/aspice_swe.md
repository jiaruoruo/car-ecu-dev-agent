# ASPICE SWE 工程过程速查（长期记忆 · 阶段映射）

V 模型左侧（设计）与右侧（测试）成对，双向追溯贯穿全程。

| 过程 | 名称 | 主要产出 | 对应 Agent |
|------|------|----------|------------|
| SWE.1 | 软件需求分析 | 软件需求规格 SRS | 需求分析 Agent |
| SWE.2 | 软件架构设计 | 软件架构 SAD + ARXML | 架构设计 Agent |
| SWE.3 | 软件详细设计与单元构建 | 详细设计 SDD + 源码 | 详细设计 / 编码 Agent |
| SWE.4 | 软件单元验证 | 单元测试规格/结果 | 单元测试 Agent |
| SWE.5 | 软件集成与集成测试 | 集成测试规格/结果 | 集成测试 Agent |
| SWE.6 | 软件合格性测试 | 合格性测试结果 | （可扩展） |

强追溯要求（SUP.10/SUP.8）：需求 ↔ 架构 ↔ 详设 ↔ 代码 ↔ 单测/集成测试 双向可追溯，
变更需走配置管理与影响分析。每个 Agent 的反馈层把对应过程的验收准则编码为质量门禁。
ISO 26262 在此之上叠加功能安全要求（ASIL 分解、安全机制、覆盖率：分支/MC-DC）。
