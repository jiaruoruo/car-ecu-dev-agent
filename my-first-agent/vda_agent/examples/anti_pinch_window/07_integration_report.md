# 07 · 集成测试报告 — 电动车窗防夹

- **过程**：ASPICE SWE.5（软件集成与集成测试）
- **产出 Agent**：集成测试 Agent（设计场景 → `hil_sil_runner` 在环执行 → `traceability`）
- **环境**：CAN-FD 在环（CANoe / dSPACE HIL）；总线 500kbit/s 仲裁 / 2Mbit/s 数据
- **集成对象**：ApwCtrl SWC + COM/PDU（CAN 信号）+ IoHwAb（PWM/ADC）

## 1. 集成测试场景

| ID | 名称 | 目标 | 步骤 | 预期 | 验证需求 | 结果 |
|----|------|------|------|------|----------|------|
| TC-IT-001 | CAN 命令→运动 | 经 CAN 下发命令驱动电机并上报状态 | 发送 PwrWinSwCmd=AUTO_UP；观测 motor_up 与 PwrWinSts | 电机上升且状态上报正确 | REQ-005,001 | ✅ pass |
| TC-IT-002 | 端到端防夹 | HIL 注入夹持力测反应时间与峰值力 | 上升中注入障碍；测反应时间与峰值力 | 反应≤100ms 且峰值力≤100N | REQ-002,003,004 | ✅ pass |
| TC-IT-003 | 堵转保护 | 模拟机械堵转 | 全闭后持续上升命令 | 进入 BLOCKED 并停止 PWM | REQ-006 | ✅ pass |
